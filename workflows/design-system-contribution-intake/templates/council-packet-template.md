# Intake packet: {proposal name}

Date, proposer, decision needed by.

## Need (from the proposal)

One paragraph restating the need in the proposer's terms.

## Overlap report

Verdict (fully covered / covered by composition / covered by small extension / partially covered / not covered), the components and docs cited, and the worked composition example if any.

## Impact report

Current occurrences of the need by product area (with file paths), the conservative adoption estimate, and the cost of doing nothing.

## Draft spec

Props, slots, and variants; semantic tokens consumed; interaction states; accessibility requirements; deferred features and why; open questions.

## Recommendation

Accept / accept narrowed / redirect to composition / decline, with confidence and the evidence each part rests on. This is the workflow's opinion, clearly labeled as such.

## Council decision (filled in by humans)

Decision, reasoning, conditions, owner, and date. This section is the record of why.
